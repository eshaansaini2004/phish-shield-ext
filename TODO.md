## Fix Plan (Prioritized)

### 1. Fix DOM results not showing in popup
**Bug.** DOM analysis runs and stores results under `dom:${url}` in chrome.storage, but `App.jsx` only reads the plain `url` key. The 10 DOM checks are silently invisible to the user.

**Files:** `src/popup/App.jsx`, `src/background/background.js`

- [x] In `background.js`, when storing the final merged result, pull in the DOM result from `dom:${url}` and merge its flags + score into the final object before writing to storage
- [x] Remove the separate `dom:${url}` write in the `DOM_ANALYSIS` message handler — everything should land in one key
- [x] `App.jsx` needs no changes once background merges correctly

---

### 2. Popup triggers analysis if no cached result
**Bug.** If you open the popup on a page already loaded (before extension ran, or cache cleared), it shows "Analyzing..." forever because `ANALYZE_URL` was never sent for that page.

**Files:** `src/popup/App.jsx`

- [x] After reading storage and getting nothing back, send `ANALYZE_URL` from the popup to background via `chrome.runtime.sendMessage`
- [x] Keep listening on `chrome.storage.onChanged` so the result appears when background finishes

---

### 3. Canvas app support (Google Docs, Figma, Notion)
**Gap.** These apps render links on canvas/custom layers — no `<a href>` tags — so hover tooltips never fire. Zero protection.

**Files:** `src/background/background.js`, `public/manifest.json`

- [x] Add `webNavigation` permission to `manifest.json`
- [x] In `background.js`, listen on `chrome.webNavigation.onCommitted` for every top-level navigation
- [x] Run URL analysis + ML on the navigated URL immediately (same pipeline as hover)
- [x] If score >= 70, show a `chrome.notifications` warning banner
- [x] Store result under the URL key so popup shows it if user clicks the extension icon

---

### 4. Retrain ML on a real phishing dataset
**Weakness.** Model was trained on synthetic data generated by the same heuristic rules — circular. Real phishing URLs won't match those patterns perfectly.

**Files:** `train_model/train.py`, `public/model/weights.json`

- [x] Add PhiUSIIL dataset (Kaggle: `shashwatwork/web-page-phishing-detection-dataset`) to `DATASET_URLS` in `train.py`, or download PhishTank verified feed CSV manually
- [ ] Run `python train.py` — verify >90% accuracy on held-out real data (not synthetic)
- [ ] Commit updated `weights.json`
